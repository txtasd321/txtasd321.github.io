IO–>标准IO 文件IO

IO

  • i–input

  • o–output

IO对象–>文件

  • 数据–>文件
  • 文件：保存在硬盘上的一组相关数据的有序集合
  • 文件名：数据集合的名称
  • Linux系统文件类型
  1. 普通文件：-
  2. 目录文件：d
  3. 字符设备文件：c–/dev——>Linux会将鼠标键盘作为字符设备文件
  4. 块设备文件：b–block——>硬盘、光盘、磁盘–>插入sd卡后在/dev目录中创建了一个块设备文件，然后访问
  5. 链接文件：l–link
  6. 管道文件：p–pipe
  7. 套接字文件：s–socket

    缓存机制

  • 标准IO库封装了缓存机制
  • 缓存：系统自动在内存中为每一个正在使用的文件开辟一个缓存区，从内存向磁盘输出数据必须先送到内存缓存区，装满缓存区再一起送到磁盘中去。从磁盘中读取数据，则一次从磁盘文件将一批数据读到内存缓存区中，然后从缓存区逐个将数据送到程序的数据区。
  • 缓存类型：
  1. 行缓存–>1024byte

全缓存区：这种缓存方式要求填满整个缓存区后才进行I/O系统调用操作。对于“磁盘文件的操作通常使用全缓冲”的方式访问。第一次执行I/O操作是，ANSI标准的文件管理函数通过调用malloc函数获得需要使用的缓存区，默认的大小为4096byte（）。

  1. 全缓存–>4096byte当缓存区满的时候就会强制把缓存区的内容刷新到shell终端

在行缓冲情况下，当在输入和输出中遇到换行符时，标准I/O库函数将会执行操作系统调用操作。当所操作的流涉及一个终端时（例如标准输入设备和标准输出设备），使用行缓冲方式。因为标准I/O库每行的缓冲区长度是固定的，所以只要填满了缓冲区，即使还没有遇到换行符，也会执行I/O系统调用操作，默认行缓冲区的大小为1024.

  1. 不缓存–>stderr(标准错误文件指针,没有缓冲区) FILE*

无缓冲区是标准I/O库不对字符进行缓存，直接调用系统调用。标准出错流stderr通常是不带缓冲区的，这使得出错信息能后尽快地显示出来。

stderr：标准错误文件指针—–>指向标准错误设备（2）
stdin：标准输入文件指针——>指向标准输入设备（0）
stdout：标准输出文件指针—->指向标准输出设备（1）

标准IO函数

  • fopen打开文件
    • r(read):打开只读文件，该文件必须存在
    • w(write):打开只写问阿金，若文件存在则文件长度清为0，即该问阿金内容会消失。若文件不存在则建立该文件。
    • a(append):以追加的方式打开只写文件。若文件不存在，则会建立该文件，即原先的内容会被保留。
    • a+：以附加的方式打开可读写文件。若文件不存在，则会建立该文件，如果文件存在，写入的数据会被加到文件尾后。即文件原先的内容会被保留。
    • t(text):文本文件，可省略
    • b(binary):二进制文件
    • r+：打开可读写的文件，该文件必须存在
    • w+：打开可读写问阿金，若文件存在则文件长度清为零，即该文件内容会、消失。若文件不存在则建立该文件。
  • fclose关闭文件
  • fread读文件size_t fread(void *prt,size_t,size_t nmemb,FILE *stream)
  • fwrite写文件size_t fwrite(const void *ptr,size_t size,size_t nmemb,FILE *stream)
  • fgets/gets读取一行
  • fputs写一行
  • fseek设置文件指针位置int fseek(FILE *stream,long offset,int whence)
  • ftell获取文件指针位置long ftell(FILE *stream)
  • freopen重定向输入输出流，即让指向一个文件流的指针指向另一个文件流
  • stdout:标准输出文件指针–>指向标准输出设备
  • stderr:标准错误文件指针–>指向标准错误设备，没有缓冲区
  • stdin:标准输出文件指针–>指向标准输入设备
  • fflush：强制将（一个流）缓冲区内容刷新到shell终端fflush(stdout);
  • setbuf将对应的缓冲设备关掉setbuf(stdout,NULL);
  • feof判断文件是否到文件末尾，0：未到文件末尾；1：到文件末尾。
  • ferror：判断对文件的读写是否出错，0：没错；非0：出错。

    文件描述符，以及它和FILE文件指针的关系

  • 文件描述符：是内核标识一个打开文件的符号，是一个非负整数，遵循最小未分配原则
  • 0 被stdin占用
  • 1 被stdout占用
  • 2 被stderr占用

文件IO函数–>Linux内核给用户提供的一组对文件的输入输出

open 打开文件

  • 参数1：要打开的文件名或带路径的文件名
  • 参数2：打开方式 O_RDONLY只读 O_WROLNY只写 O_RDWR读写
  • 参数3：给文件赋予的文件权限（八进制0666），实际该文件的权限是=0666&(~umask)
  • 返回值：正确，int数—>文件描述符（从3开始向后） 错误：-1.

close 关闭打开的文件

  • 参数1：文件描述符

read 读取打开文件的内容

  • 参数1：文件描述符
  • 参数2：本地准备好的存放读到内容的buf
  • 参数3：期望读取的元素个数
  • 返回值：>0：成功读到的元素个数。 =0：读到文件末尾。 <0：出错

write 往文件中写入内容

  • 参数1：文件描述符
  • 参数2：本地准备好内容（存放在buf中）
  • 参数3：期望写入的元素个数
  • 返回值：>0：成功写入文件的字节数 <=0：错误（0：写入内容为0；-1：写入时出错

lseek： 设置文件指针的位置，不能设置管道S I ,可以设置：- d c b p l s

函数库及制作

函数库—>存放了大量的函数的二进制文件(无main)，该文件是别人已经写好的。
Linux系统中，有两种函数库：静态库 动态库
静态库：.a
动态库(共享库)：.so
用户使用库的时刻不同：编译时—–> .a 运行库—–> .so
.c—–>编译——>./a.out——->运行
特点：
静态库：体积变大，移植方便
动态库：体积不会变大，多个应用程序可以使用同一个动态库，移植不方便。
使用场合：
应用程序代码量较小，建议使用静态库
应用程序代码量较大，建议视同动态库

库的制作步骤

静态库：

  1. 首先写函数库源码func.h func.c和测试源码main.c
  2. 编译func.c为func.o文件 gcc -c func.c -o func.o
  3. 创建静态库，将很多.o转化成.a ar crs libmyfunc.a func.o
  4. 编译自己的程序使用静态库：gcc -o main main.c -L. -lmyfunc
  5. 测试

动态库：

  1. 首先写函数库源码func,h func.c 和测试代码main.c
  2. 编译func.c为func.o文件 gcc -fPIC -Wall -c func.c 注：-fPIC：创建与地址无关的编译程序 -Wall：生成所有警告信息
  3. 创建动态库 gcc -shared -o libmyfunc.so func.o
  4. 将生成的动态库拷贝到/lib目录下 sudo cp libmyfunc.so /lib/
  5. 编译自己的程序使用静态库：gcc -o main -L. -Lmyfunc.so
  6. 测试
