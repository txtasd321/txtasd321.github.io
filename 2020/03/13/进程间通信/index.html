进程间的通信——>一台主机上两个进程间的通信

ulimit 控制shell程序的资源
-a 显示目前资源限制的设定
-c <core文件上限> 设定core文件的最大值，单位Wie区块
-d<数据节区大小> 程序数据节区的最大值，单位为KB。
-f <文件大小> 　shell所能建立的最大文件，单位为区块。
-H 　设定资源的硬性限制，也就是管理员所设下的限制。
-m <内存大小> 　指定可使用内存的上限，单位为KB。
-n <文件数目> 　指定同一时间最多可开启的文件数。
-p <缓冲区大小> 　指定管道缓冲区的大小，单位512字节。
-s <堆叠大小> 　指定堆叠的上限，单位为KB。
-S 　设定资源的弹性限制。
-t <CPU时间> 　指定CPU使用时间的上限，单位为秒。
-u <程序数目> 　用户最多可开启的程序数目。
-v <虚拟内存大小> 　指定可使用的虚拟内存上限，单位为KB。

  1. 信号通信

信号通信是在软件层面上对中断的一种模拟
中断：在程序执行的过程中插入了另一段程序的执行过程，该段程序执行完后，回到中断点继续原来的程序。

信号–>64个信号

异步通信：通信中接收方并不知道信号(数据)什么时候会到达，当前进程一直准备着接受信号(数据)同时也在做自己的事情，一旦信号(数据)到达，接受处理。所以是非阻塞模式。

同步通信：发送方发数据，接收方接数据，双方需要在很短的时间内完成数据交换，否则会造成一方数据阻塞。所以是阻塞模式

linux中的常见信号：

信号名 含义 默认操作
SIGHUP 该信号在用户终端连接（正常或非正常）结束时发出，通常是在终端的控制进程结束时，通知同一会话内的哥哥作业与控制终端不再联系 终止
SIGINT 该信号在用户INTR字符（通常是ctrl+c）时发出，终端驱动程序发送此信号并送到前台进程中的每一个进程。 终止
SIGQUIT 该信号和SIGINT类似，但由QUIT字符（通常是Ctrl-\）来控制。 终止
SIGILL 该信号在一个进程企图执行一条非法指令时(可执行文件本身出现错误，或者师徒执行数据段、堆栈溢出时)发出 终止
SIGFPE 该信号在发生致命的算数运算错误时发出。这里不仅包括发电运算错误，还包括溢出及除数为0等其他所有的算数的错误 终止
SIGKILL 该信号用来立即结束程序的运行，并不能被阻塞、处理和忽略 终止
SIGALRM 该信号当一个定时器到时的时候发出。 终止
SIGSTOP 该信号用于暂停一个进程，且不能被阻塞、处理、忽略 暂停进程
SIGTSTP 该信号用于暂停交互进程，用户可键入SUSP字符（通常是Ctrl-Z)发出这个信号 暂停进程
SIGCHLD 子进程改变状态时，父进程会受到这个信号 忽略
SIGABORT 该信号用于结束进程 终止

进程对信号的处理方式：忽略 捕获 采用默认处理
命令|对应信号|默认（缺省）动作|功能
-|-|-|-
alarm(2)定时器功能|SIGALRM|终止当前进程|alarm是一个闹钟函数，可以在进程中设置一个定时器，当定时器指定时间到时，内核就向进程发送SIGALRM信号
pause()||挂起当前进程|等待一个信号的到来

SIGALRM对应信号为终止，改信号对应的默认处理动作
signal–>修改一个信号对应的默认动作
参数1：信号名
参数2：函数(新的处理动作) void func(int signo)

注意：如果当前进程使用了函数，那么当前进程会有一次进入内核空间拿数据的机会，如果在进程中使用了信号，那么当前进程会有两次机会进入内核空间

kill()和raise()
kill可以发送信号给进程或进程组（实际上，kill系统命令指示kill函数的一个用户接口）
kill -l 命令查看系统支持的信号列表
raise函数允许进程向自己发送信号

kill:
函数表头：
#include <signal.h>
#include <systypes.h>
函数原型：
int kill(pid_T pid,int sig);
函数参数：
pid: 正数：要接受信号的进程的进程号 ； 0：信号被发送到所有和pid进程在同一进程组的进程；-1：信号发给所有的进程表中的进程（除了进程号臭的进程外）

sig:
信号
函数返回值：成功：0；失败：-1

自定义异步通信：

  1. a 进程接受信号 打印自己的pid

  2. b进程 发信号–>kill(pid,信号) pid–>main函数传参来 获取pid–>调用kill(pid,SIGUSR1)

  3. 发信号：kill raise

  4. 收信号：pause
