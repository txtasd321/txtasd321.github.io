守护进程与多线程的应用

守护进程

定义：周期性的执行某项任务或等待某个时间发生的进程，它的运行不依赖shell终端，他的生存较长，从开机开始到关机结束。
ps -auj：可以查看，它的TTY是？，代表不知终端是哪个，就是守护进程。
如何创建守护进程？

  1. 创建子进程，让父进程退出—–>fork pid >0 exit
  2. 设置新的会话期—>此时守护进程就创建好了—->setsid()
  3. 设置工作目录为根目录/ —–>
  4. 设置文件权限掩码—->umask(0)
  5. 关闭所有文件描述符—–>getdtablesize() close

线程

定义：线程是程序的一次执行过程，线程共享创建它的进程的地址空间。
进程 = task_struct + 4G地址空间
线程 = task_struct + 地址空间（此地址空间是创建该线程的进程的）
线程也统一参与操作系统的调度
/.a.out——>函数

task_struct：

  1. 标识符：
    pid—->进程ID
    tgid—–>线程组ID
  2. 状态
  3. 优先级
  4. 。。。。。。

tid—->标识线程的符号，就是线程ID
情况1：当进程中午线程时，tgid = pid
情况2：当线程中有多个线程，tgid标识主线程的id而pid表示每个线程的ID，即tid。

原进程tgid = pid； 线程：pid

如何创建线程？调用第三方线程库

int pthread_create(pthread_t *thread,const pthread_attr_t *attr,void*(*start_routine) (void *),void *arg);//创建线程

表头文件：#include<pthread.h>
参数1：线程ID
参数2：线程属性，一般给NULL，使用默认属性
参数3：线程执行函数
参数4：传给线程执行函数的入参
返回值：成功返回 0，失败返回其他值。

pthread_join()—>等待进程退出,处理线程退出状态值
参数1：线程ID
参数2：NULL，表示不关心线程退出状态，ptr，子进程退出时传给主进程的退出状态值

pthread_exit()：线程退出函数，在执行函数中调用
参数1：线程退出状态（一级指针表示）

pthread_cancel():取消线程
参数1：线程ID

系统自启动文件在/etc//init.d/sshd
用户自启动文件在/etc/rc.local

多线程的引入

  1. 线程间数据交换——>全局变量
  1. 线程的同步：两个线程相互配合完成一件事情—->信号量（无名信号量）—–>flag = 0—–>线程1执行， = 1 —–>线程2执行
    同步：两个对象相互配合完成一件事
    用信号量（二值信号量0/1）sem_t sem1 = 0/1 0:不能使用资源 1：可以使用资源

sem_init():初始化信号量
参数1：要初始化的信号量
参数2: 0：线程间使用，非0：进程间使用
参数3：赋给参数1的值（0/1）
返回值：0：成功； -1：失败

P操作：sem_wait(sem1):如果检测到sem1>0,函数返回，解除阻塞，并且sem1-1，如果sem1=0,该函数阻塞当前进程

V操作：sem_post(sem2):sem2+1

  1. 线程的互斥（多线程对公共资源的竟态访问）——加线程锁
    线程1：func—>公共区域
    线程2：func
    例子：两个线程，func—->static char buf[]
    互斥锁：

  2. pthread_mutex_t mutext
    pthread_mutex_int 初始化锁
    参数1：互斥锁
    参数2：互斥锁属性，NULL表示缺省属性

  3. pthread_mutex_lock（&mutext）锁定线程
    参数1：互斥锁

  4. pthread_mutex_unlock(&mutext) 对锁定线程解锁
    参数1：互斥锁
