gdb的使用

在linux程序调成中gdb是最常用的工具。

关于gdb的说明可以在linux下直接 man gdb或者gdb –help。
gdb的目的主要是让你知道程序内部正在运行什么代码，或者是当程序崩溃的时候正在执行什么代码

GDB主要可以做四种事情来帮助你找到bug：
1.启动你的程序，指定任意可以影响程序行为的参数。
2.让你的程序在指定的条件停住.
3.测试你的程序停止的时候发生了什么。
4.改变程序内部的变量，来改正程序的错误继续执行。

首先安装gdb

sudo apt install gdb

vim test.c
编译：gcc -g test.c -o test
运行 gdb test
参数如下：
编号|命令缩写|命令说明
-|-|-
1.|l|使用l来查看源码（一次查看10行）|
2.|设置断点 b|(breakpoint) main或者 b 行号
3.|info b |查看断点信息
4.|del n(n为断点编号)| 删除断点
5.|q|按q退出
6.|r(run) |在断点后跑起来
7.|n(next)| 继续执行(有子函数不进入，仅当一句话执行)
8.|p |变量名 查看变量信息
9.|s(step)| 继续执行(有子函数，进入子函数)
10.|c| 调到下一个断点
11.|k |终止正在调试的程序
12.|f|查看栈帧
13.|st|开始执行程序，在main函数的第一条语句前面停下来
14.|disp|跟踪查看某个变量，每次停下来都显示它的值
