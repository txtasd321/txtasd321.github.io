Linux_bash

shell基本命令
语法格式 command [-p] arg1 arg2 …

命令名称   选项    参数
注意：
    1.命令名称一定不能少，选项和参数是可有可无的
    2.多条命令用分号隔开（；）
    3.跨行输入命令后面加反斜杠\    （要保证命令的正确性，特别是空格）

vim passwd
用户：密码：ID：组ID：工作目录:bash

adduser

deluser 不完全删除
deluser –remove-home xxx 完全删除

huiam
hsotname

ls -li 显示当前文件信息，包括inode号

cat 显示文件内容
cat -n 加行号

echo 输出
echo “ ” 原样输出

chmod 改变权限
1.八进制 000~111 rwx
2.ugo a +/-

grep 在指定路径中查找指定内容
“linux” /etc/passwd

ln 连接命令（实质上就是一种快捷方式，保留目标文件的地址）
软连接（符号连接）：通过文件的路径建立链接
ln -s target(目标文件名) link_name(生成的文件名) target最好使用绝对路径
硬链接：通过文件的物理编号建立—-inode链接
ln 文件名 hlink_name

压缩打包
归档：对文件进行整理打包(对文件的大小没有改变)
tar(对文件进行压缩，文件大小有改变。)
-c 建立新的打包文件
-r 项打包文件末尾追加文件
-x 从打包文件中解压出文件
-o 将文件解开到标准输出
-v 处理过程中输出相关信息
-f 对普通文件操作
-z 调用gzip来压缩打包文件，与”-x”联用时调用gzip完成压缩
-j 调用bzip2来压缩打包文件，与”-x”联用时调用bzip2完成压缩
-Z 调用compress来压缩打包文件，与”-x”联用时调用bzip2完成解压缩
//什么方式压缩，什么方式解开 压缩包后面把后缀名带上

tar -czvf  目标文件    新文件名     //打包并压缩文件
    在ubuntu18中目标文件与新文件名位置相反

tar -xjvf  目标文件        //解压文件

shell中的特殊字符

1.通配符 
     *   匹配任意长度的字符    file*
     ？    匹配一个长度的字符     file?
     [abcd]    匹配其中一个字符    fiel[abcd]
     [ - ]     [a-z]    匹配a-z其中一个字符    file[a-z]
     [^]     [^1-9]    匹配除了1-9中的字符    file[^1-9]


2.管道   |    :连接作用，将第一个命令的输出作为第二个命令的输入，以此类推
    cat /etc/passwd | grep "bash"


3.wc
    计算文件的行数、单词数、大小
    计算目录下文件的个数

4.ps    查看进程
    ps -ef | grep "bash"    查看开了几个bash

5.输入输出重定向
    默认输出输出源：

    输出
        >file_name        (只能输出正确)的将该文件重定向为输出源，新建模式
            ll >tmp1.txt
        >>file_name        (只能输出正确的)在原有的文件中继续写入，没有则新建，追加模式
        2>file_name        错误信息的输出的源重定向，新建模式
        2>>file_name        错误信息的输出的源定向，追加模式
        &>file_name        (无论对错)只要是命令结果，都保存在file_name
        &>>file_name    (无论对错)只要是命令结果，都保存在file_name，追加模式

    输入
        <file_name        

        cat :直接cat，使用默认输入源（键盘）   输出源是当前屏幕
        cat >1.txt：输入源是默认的，输出源是1.txt
        cat <1.txt  >2.txt :输入源是1.txt 输出源是2.txt



命令置换：将一个命令的输出作为另一个命令的参数
    ls `pwd` (esc下方的点)
