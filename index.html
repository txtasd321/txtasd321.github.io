动态内存

动态内存管理

malloc：
void malloc(size_t size)
char *p = (cahr *)malloc(szieof(char)10);
free

1.malloc不关心内存保存什么数据类型，只关心申请的大小
2.在使用这片空间之前，一定要强制转换
3.如果申请内存成功，malloc会返回这片连续内的首地址，失败返回NULL
4.一定要显式释放这片内存    free()
5.释放之后，把指针赋值为NULL，避免空悬指针
6.只申请不释放，会造成内存泄漏
7.不能重复释放这片内存

linux_shell_脚本流程

shell变量
shell是没有数据类型的

1.用户自定义变量
    从右到左赋值，
    赋值运算两边不能有空格，
    引用shell比那辆的值一定要加$
    Y=y   X=$Y   echo $X   y
    ubset X //删除X的值，echo $X  为空

2.位置变量(命令行的参数)
    $0    脚本文件名
    $1 ~ $9    命令行的第一个参数 ~ 第九个参数
    ${10}    命令行的第十个参数

3.预定义变量
    $#    命令行参数的个数(不包含脚本文件名)
    $@    所有的命令行参数
    $*    所有的命令行参数
    $?    上一行命令若果结果是0  代表执行成功     为1则失败

4.环境变量
    PATH        标准环境变量
    HOME        查看HOME
    SHELL        查看SHELL
    HISTSIZE    历史命令的个数

    env    export    两个变量都是查看环境变量

    \        转移字符



shell语句
1.说明性语句    以 # 开头   #！/bin/bash
2.功能性语句
    1、read：从标准输入读一行赋给shell变量
    read VAR1  VAR2  VAR3。。。
    echo    $VAR1    $VAR2    $VAR3

    expr 3 + 2 \* 6        算数运算

    2、expr ( + - \* / \( \))    整数算数运算命令
    read A
    expr $A + 2 \* 6
    NUM=`expr 6 + \( 5 \* 5 \)`
    echo $NUM
    echo `expr $NUM \* 6`     
    注意：
        乘号和小括号要加转移符
        运算两边要留有空格
        若果把算数的结果赋给变量要用命令置换
    3、test     测试对象：整数    字符串    文件类型
        整数测试
        a -eq b     判断是否a是否等于b
        a -ne b        判断是否a和b不相等
        a -gt b        判断是否a大于b
        a -lt b        判断是否a小于b
        a -ge b      判断是否a大于等于b
        a -le b        判断是否a小于等于b

        测试字符串
        s1 = s2        a等于b
        s1 != s2    a不等于b
        测试的时候两边要留有空格
        test -z    s1  测试长度
        文件类型
            -    普通文件
            d    文件夹
            l    链接文件
            c    字符设备文件
            b    块设备文件
            p    管道文件
            s    套接字文件

        文件类型测试
            -d file_name 该文件是否是文件夹
            -f file_name 该文件是否是普通文件
            -S file_name 套接字文件
            -L file_name 连接文件
            -c file_name 字符快文件
            -b file_anme 块文件
            -p file_name 管道文件
            -e file_name 文件是否存在

-r 项打包文件末尾追加文件

-v 处理过程中输出相关信息

-j 调用bzip2来压缩打包文件，与”-x”联用时调用bzip2完成压缩

shell中的特殊字符

1.通配符 
     *   匹配任意长度的字符    file*
     ？    匹配一个长度的字符     file?
     [abcd]    匹配其中一个字符    fiel[abcd]
     [ - ]     [a-z]    匹配a-z其中一个字符    file[a-z]
     [^]     [^1-9]    匹配除了1-9中的字符    file[^1-9]


2.管道   |    :连接作用，将第一个命令的输出作为第二个命令的输入，以此类推
    cat /etc/passwd | grep "bash"


3.wc
    计算文件的行数、单词数、大小
    计算目录下文件的个数

4.ps    查看进程
    ps -ef | grep "bash"    查看开了几个bash

5.输入输出重定向
    默认输出输出源：

    输出
        >file_name        (只能输出正确)的将该文件重定向为输出源，新建模式
            ll >tmp1.txt
        >>file_name        (只能输出正确的)在原有的文件中继续写入，没有则新建，追加模式
        2>file_name        错误信息的输出的源重定向，新建模式
        2>>file_name        错误信息的输出的源定向，追加模式
        &>file_name        (无论对错)只要是命令结果，都保存在file_name
        &>>file_name    (无论对错)只要是命令结果，都保存在file_name，追加模式

    输入
        <file_name        

        cat :直接cat，使用默认输入源（键盘）   输出源是当前屏幕
        cat >1.txt：输入源是默认的，输出源是1.txt
        cat <1.txt  >2.txt :输入源是1.txt 输出源是2.txt

结构体类型

结构体类型

struct date
{
int year;
int month;
int day; //成员变量之间用分号隔开，成员变量之间数据类型可以相同也可以不同
}

struct date 才是结构体数据类型，struct只是一个关键字，date只是结构体名，此时没有分配空间

d1是一个局部变量，d2是一个全局变量

struct date d1 = {2020, 1, 13}
访问结构体成员变量名：
结构体变量名.成员名 d1.year d1.month d1.day

结构体变量之间是可以直接进行赋值的：

1、无名结构体：结构体名

2、结构体指针

3、结构体数组

通过指针访问结构体成员
指针名->成员变量名

4、字节对齐（32os)
系统会一次性分配4个字节
double一次性分配4个字节，分两次分配

5、共同体

==注意==
1.所有成员变量的首地址一致
2.共同体变量的内存空间取决于成员内存最大的成员
3.耨遗失可，只能取出一个变量来

6、判断计算机大小端序
大端序：低地址存高数据
小端序：高地址存高数据

7、枚举

给成员赋值时只能赋值为整形数
一旦成员的值确定，就不能再进行赋值
如果不给第一个初始化，默认值为0，后面的每个值加1
